# FindManager 1.0.0-rc1

## 概要

`1.0.0-rc1`は、工程1から工程13までの主要業務機能と、工程14の本番導入準備を統合した最初のリリース候補です。正式な本番デプロイや実データ投入を行うリリースではありません。

## 主な機能

- セッション認証、CSRF、スタッフ・ロール・監査管理
- 拠点、業務区分、スタッフ配属・能力マスタ
- シフトパターン、週間テンプレート、月間シフト、15分タイムライン、印刷
- シフト確定・公開snapshot、本人閲覧、希望休、公開後変更申請
- 打刻、休憩、退勤、修正申請、勤務実績確定、月次勤怠締めとCSV
- 勤務単価・手当、概算人件費、人件費予算、予定原価、予実差異
- 売上区分、売上予算・実績、売上達成率、人件費率、月次経営snapshotとCSV
- 環境変数ベースの本番設定、health/readiness、request ID、構造化ログ
- システム状態画面、Playwright E2E、強化CI、最小Docker構成
- 初期導入、日次・月次運用、バックアップ・復元、障害対応、リリース手順

## 既知の制約

- 金額は運営管理上の概算・予実値であり、正式給与、会計、税務の確定値ではありません。
- 会員管理、SLIM、銀行、会計・給与ソフト、外部通知との自動連携はありません。
- 売上実績は手入力を基本とし、PDF・Excel出力はなく、CSVのみです。
- 法定割増、税・社会保険、休暇残高、仕訳、請求・入金消込は対象外です。
- 本番監視、メール、外部ストレージ、証明書、DNS、secret管理基盤は導入環境で選定します。
- 高人件費率warningは40%の固定運用閾値で、会計規則や自動制御ではありません。
- `check --deploy --tag security`はcleanですが、全tagのschema checkには既存SerializerMethodField等のOpenAPI型注釈warningが残ります。runtime APIと権限は自動testで検証済みです。

## 導入前チェック

1. [release-checklist.md](../operations/release-checklist.md)を完了する。
2. 本番秘密値、HTTPS、許可host、CSRF origin、secure cookie、HSTSをレビューする。
3. `check --deploy`と`check_deployment_readiness`を本番相当設定で成功させる。
4. 本番と同等のPostgreSQLへ復元リハーサルを実施する。
5. 管理者、ロール、拠点、必須マスタ、初期パスワード状態を確認する。
6. ステージングで主要E2Eと手動smoke testを実施する。

## Rollback概要

リリース前にDB・media・現在のimage/tagを保存します。障害時は新規流入を停止し、アプリimageを直前のtagへ戻します。後方互換性のないmigrationを伴う場合は、承認済み手順でDBをリリース前バックアップへ復元し、`migrate --plan`、readiness、snapshot整合性、主要smoke testを確認してから再開します。データを保持したままコードだけ戻せるかはmigrationごとに事前判断します。
